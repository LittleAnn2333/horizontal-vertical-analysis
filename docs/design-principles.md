# 设计原则与方法论

这份文档解释项目背后的几个核心设计决策。理解这些原则有助于使用、扩展或贡献本项目。

---

## 原则一:这是一个**框架**项目,不是**数据接入**项目

很多人以为做股票研究 skill 的难点是"接入金融数据 API"。我们的判断恰好相反——**信息收集的难度已经被金融工具解决了,真正难的是把数据加工成能讲清楚因果、能给出判断、能传递给同事的产物**。

所以本项目不提供任何数据接入能力。它解决的是"用户已经有数据后,如何结构化成研究报告"这一段。

### 这意味着什么

- 我们不需要 Wind / Choice / Bloomberg 的 API
- 我们不需要复杂的 ETL 流程
- 我们不需要在持续维护数据源更新

我们需要的是:**一套能让任何一份数据被加工成有判断力的报告的框架**。

---

## 原则二:融合"叙事力"与"金融严谨"

研究报告有两种典型失败模式:

### 失败模式 A:太干瘪(像研报数据库)

```
公司 2024 年营收 21.94 亿,同比增长 31.42%。毛利率 21.16%。
```

数字都对,但没人想读。这种报告本质是"数据列表",对读者没价值。

### 失败模式 B:太空(像公众号软文)

```
公司近年来发展迅猛,业绩表现亮眼,展现出强劲的成长性。
```

读起来流畅,但什么都没说。这种报告本质是"营销话术"。

### 我们要的:既有数字又有叙事

```
2022 到 2024 三年营收翻了 2.4 倍(从 9.19 亿到 21.94 亿),
2025 增速明显放缓至 9.2%。这是一条很经典的"S 型增长曲线右半边"——
锂电铜箔扩产最热那一波已经过去,但有一个数据藏在底下容易被忽略:
电子电路铜箔设备占比从 2022 年 10.9% 悄悄爬到了 2025 年 34.39%。
公司在锂电最热时已经在偷偷调头转向 PCB 市场。
```

这种写法的特征:**具体数字 + 因果链 + 一个"藏在数据下的发现"**。

---

## 原则三:不出具体目标价,但要给可调试的估值算式

很多研究报告的最终结论是"目标价 XX 元"——这个数字本质是营销话术,因为它把多个假设打包成一个不可拆解的输出。

本项目坚持的做法是:**给三剧本概率分布 + 每个剧本的估值锚算式**。

例如:

```
Base case 概率 55-65%
估值锚:2027E 归母 40.94 亿 × 50-60 倍 PE = 市值 2047-2456 亿
```

这种写法的好处:

- 用户可以代入不同的 PE 倍数自己算结果
- 用户可以质疑"50-60 倍 PE 假设是否合理"
- 用户可以替换 EPS 假设看结果
- 这是**可调试的估值**,不是不可质疑的目标价

## 原则四:诚实标注数据缺口和推断质量

研究报告里经常有这种话:

```
公司客户集中度较高,前五大客户大约占 50% 左右。
```

"大约"、"约"这种词汇本质是把"我不知道"装扮成"我估了一下"。这种语言会让读者无法识别哪些是真数据、哪些是推测。

本项目坚持的做法是两层显性标注:

### 第一层:正文里直接标注

```
现有资料未给出前五大客户名单和收入占比,因此无法进一步精确计算客户集中度。
从行业惯例推断,基础设施服务类公司前五大客户占比常在 40-60% 区间。
```

### 第二层:打折清单里量化标注

每个论点的可信度被打折(0-9 折)+ 重要性标签(🔴 支柱 / 🟡 辅助 / 🟢 背景)。

读者一眼能看出报告的"支柱论点是不是扎实"。

---

## 原则五:让用户参与推理过程

很多 AI 工具的产出方式是"一键生成完整报告"。这看起来高效,但实际上把用户排除在了推理过程之外。

本项目的工作流强制要求 **Step 3.5 骨架预览**——在生成完整报告之前,把核心判断先给用户看,让用户在最低成本的时间点介入修改。

骨架包括"最有可能出错的 2 个判断"——强迫 Claude 承认推理链中最脆弱的环节,让用户精准定位需要讨论的地方。

这一步把工具从"自动化"升级为"协作"。

## 原则六:横纵分析框架

横纵分析框架是本项目的方法论核心,融合了两个思想源流:

- **历时-共时分析(Saussure 语言学)** — 任何对象都可以从"时间维度"和"截面维度"两个角度理解
- **商学院案例研究 + 竞争战略** — 把单一公司放进产业链和竞争格局里看

### 纵向分析(Diachronic)

核心问题:这家公司**为什么长成今天这个样子**?

不是简单的"回顾历史",而是要识别**关键决策节点**和**路径依赖**:

- 哪一年发生的事?
- 当时是什么决策?
- 这个决策为什么后来变成了不可逆的路径依赖?
- 如果没做这个决策,今天会是什么样?

### 横向分析(Synchronic)

核心问题:这家公司**当下在产业链里占什么位置**?跟谁竞争?跟谁对标?

三层对手分析法:
1. **直接产品竞争对手**
2. **上下游延伸者**(真正的威胁往往在这一层)
3. **估值锚定板块对照组**

### 横纵交汇(Intersection)

核心问题:历史决策路径如何**塑造了今天的位置**?未来怎么走?

具体动作:
- 把今天的每个优势/劣势挖到具体的历史决策上
- 三剧本概率分布(不出目标价,出概率分布)

详细的框架定义见 `stock-deep-research/references/framework.md`。

## 这些原则一起服务于一个目标

让任何使用本 skill 的人,能产出一份**既好读、又严谨、又透明、又能讨论**的研究报告。

不靠数据壁垒,不靠模型黑魔法,只靠一套被精心设计的框架。
