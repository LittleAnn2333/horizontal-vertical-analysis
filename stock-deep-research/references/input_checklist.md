# 用户输入素材清单 (Input Checklist)

本文档说明:**做一份完整的深度研究报告,理想情况下需要哪些素材**。用户根据这份清单准备素材,Claude 会对照清单做"素材体检"。

---

## 数据维度总览

| 维度 | 最低要求 | 理想要求 | 用于哪个章节 |
|---|---|---|---|
| 公司基本面 | 主营、实控人 | 股权结构、子公司、历史沿革 | 一句话定义 / 纵向分析 |
| 历史时间线 | 关键转型节点 | 详细年表 + 决策背景 | 纵向分析 |
| 财务数据 | 近 3 年三表关键指标 | 近 5 年 + 分板块毛利率 | 业务与财务底盘 |
| 业务结构 | 主营板块划分 | 分板块收入/毛利率/趋势 | 业务与财务底盘 |
| 客户/供应商 | 主要客户名单 | 集中度、新拓客户、海外客户 | 业务与财务底盘 |
| 订单与产能 | 在手订单总额 | 订单结构、产能利用率、发货节奏 | 业务与财务底盘 |
| 同业数据 | 1-2 家直接竞争对手 | 3 类对手(直接/延伸/估值锚) | 横向分析 |
| 估值数据 | 当前 PE/PB | 历史分位、可比公司估值大表 | 横向分析 / 横纵交汇 |
| 一致预期 | 1-2 篇研报观点 | 3-5 家卖方目标价/盈利预测 | 横纵交汇(三剧本) |
| 业务催化 | 最近半年公告/订单 | 完整事件时间线 + 卖方核心逻辑 | 横纵交汇(三剧本) |
| 风险点 | 行业基本风险 | 公司特有风险 + 财务风险 | 风险提示 |

## 素材组织建议

### 推荐的提交方式

用户可以用以下任一方式提交素材,**Claude 都能处理**:

1. **直接粘贴文本**:把研报、公告、新闻摘要直接贴在对话里
2. **上传 PDF 文件**:招股书、研报、年报
3. **上传 Word/Excel**:金融工具导出的"公司一页纸"、数据表
4. **上传图片/截图**:网页、表格截图
5. **混合方式**:以上几种组合

### 建议的素材分类

为了让 Claude 高效处理,建议用户在提交素材时大致分类:

```
1. 公司基本资料(公告、年报、招股书摘要)
2. 财务数据(关键指标、分板块数据)
3. 业务与订单(在手订单、客户名单、产能)
4. 同业数据(竞争对手财务、市占率)
5. 估值与预期(估值表、卖方研报)
6. 近期催化(公告、新闻、订单)
```

不分类也行,Claude 会自己梳理。但分了类素材体检更快。

---

## "够用"的最低门槛

如果时间紧或者素材有限,以下是"勉强够写"的最低数据:

- 公司近 3 年营收+净利+毛利率
- 主营业务 1-2 段描述
- 至少 1 家直接竞争对手的财务数据
- 当前估值(PE/PB)
- 最近的 1-2 个催化事件

低于这个门槛,报告会出现大量"数据不足"的标注,可读性下降。Claude 会在素材体检阶段告知用户。

---

## 关于"信息时效"

不同信息的时效要求不同:

| 信息类型 | 时效要求 |
|---|---|
| 财务数据 | 最新一份季报/年报为准 |
| 估值数据 | 越新越好(理想:最近 1 周内) |
| 同业对比 | 同一时点对比(避免不同时点混用) |
| 历史决策 | 可以是几年前的,但要标注年份 |
| 行业事件 | 最近 12 个月内的 |

用户提供素材时如果是旧数据,Claude 会在报告里注明"截至 XX 时点"。
